## 1. 프로젝트 개요

- 프로젝트명: AgentMark
- 한 줄 설명: 긴 텍스트를 구조화된 형태로 변환하는 요약 자동화 도구

본 프로젝트는 비정형 텍스트를 일정한 구조로 정리하는 자동화 도구를 구현하는 것을 목적으로 한다.

단순 요약을 넘어, 핵심 정보가 일정한 형식으로 정리되어 출력되도록 하는 데 초점을 둔다.

출력 결과는 사람이 빠르게 이해할 수 있도록 항목화된 형태(예: 섹션 기반 요약, 마크다운 구조 등)로 제공하는 것을 지향한다.

---

## 2. 목표

본 프로젝트는 AI를 활용하여 긴 텍스트를 구조화된 형태로 변환하는 자동화 기능을 구현하는 것을 1차 목표로 한다.

특히, 비정형 텍스트를 입력으로 받아 핵심 정보를 분리하고, 이를 일정한 구조로 재구성하는 과정의 자동화를 목표로 한다.

또한, 프롬프트 설계 및 후처리 전략에 따라 결과 품질이 어떻게 달라지는지를 실험하고 검증하며,

단순 기능 구현을 넘어 결과의 안정성과 재현성을 확보하는 것을 핵심 목표로 한다.

---

## 3. 과제 선택 이유

기존에 AI를 활용한 기능 구현 프로젝트를 진행하며, 모델을 활용한 결과 생성에는 익숙해졌지만,
이를 실제 서비스 형태로 자동화하는 경험은 부족하다고 느꼈습니다.

AI를 단순히 사용하는 수준을 넘어 실제 자동화 도구 형태로 구현하고,
그 과정에서 발생하는 문제를 직접 해결해보고자 본 과제를 선택하였습니다.

특히, 자동화 과정에서 결과의 일관성과 품질을 어떻게 안정적으로 유지할 수 있는지를 직접 검증해보고자 하였습니다.

---

## 4. 핵심 기능 및 선정 이유

- 사용자 입력 (비정형 텍스트)
    - 자동화 프로세스의 출발점 확보
- 구조화 결과
    - 핵심 요약: 전체 내용의 가독성 및 전달력 향상
    - 키워드 추출: 데이터 분류 및 빠른 정보 검색 지원
    - 행동 포인트: 정보 소비 후 즉각적인 실행 가이드 제시
- 구조화 변환
    - Markdown 또는 표(Table) 형식 제공: 정보의 가시성 및 체계적인 저장 환경 구축

---

## 5. 기술 스택 및 선정 이유

FastAPI (Python) 

- 현재 가장 익숙한 기술 스택이며, FastAPI의 비동기(Async) 처리 방식은 AI API 응답을 기다리는 자동화 도구의 효율성을 높이는 데 적합하다고 판단했습니다.

Pydantic

- AI의 응답 형식이 일관되지 않을 때 발생하는 코드 에러를 방지하고, 데이터 구조의 안정성을 확보하기 위해 도입했습니다.

LangChain

- 복잡한 파이프라인 설계를 간소화하고, 프롬프트 템플릿 관리를 통해 유지보수 효율을 높이고자 선택했습니다.
